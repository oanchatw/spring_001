# 快速啟動指南

## 前置需求
- Java 17+
- Maven 3.8+
- Docker Desktop（已安裝）
- Google Cloud Console 帳號

---

## Step 1：取得 Google API 金鑰

### Google OAuth2 (登入用)
1. 前往 https://console.cloud.google.com/
2. 建立新專案
3. 左側選單 → APIs & Services → Credentials
4. 建立「OAuth 2.0 Client ID」
   - 類型：Web application
   - 授權的重新導向 URI：`http://localhost:8080/login/oauth2/code/google`
5. 記下 **Client ID** 和 **Client Secret**

### Google Maps API Key (地圖功能用)
1. 同一個專案 → 建立「API Key」
2. 啟用以下 API：
   - Maps JavaScript API
   - Places API
   - Directions API
   - Geocoding API

---

## Step 2：啟動 PostgreSQL（Docker）

```bash
cd spring-map-app

# 背景啟動 PostgreSQL 容器
docker compose up -d

# 確認容器狀態（應顯示 healthy）
docker compose ps
```

容器啟動後，PostgreSQL 即在 `localhost:5432` 運行，資料庫名稱 `mapapp_db` 已自動建立。

---

## Step 3：設定 Google API 金鑰

**方法 A：直接修改 application.properties**（最簡單）
```properties
spring.security.oauth2.client.registration.google.client-id=1234567890-xxxx.apps.googleusercontent.com
spring.security.oauth2.client.registration.google.client-secret=GOCSPX-xxxxx
google.maps.api-key=AIzaxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**方法 B：環境變數（推薦，避免金鑰外洩）**
```bash
export GOOGLE_CLIENT_ID=1234567890-xxxx.apps.googleusercontent.com
export GOOGLE_CLIENT_SECRET=GOCSPX-xxxxx
export GOOGLE_MAPS_API_KEY=AIzaxxxxxxxxxxxxxxxxxxxxxxxxxx
```

---

## Step 4：啟動 Spring Boot

```bash
mvn spring-boot:run
```

啟動成功後：
- 首頁（登入頁）：http://localhost:8080
- Hibernate 會自動建立 `users`、`saved_places`、`routes` 三張資料表

## API 測試範例（curl 或 Postman）

> 注意：需要先用瀏覽器完成 Google 登入，取得 Session Cookie 後再測試 API

### 取得當前用戶
```
GET http://localhost:8080/api/auth/me
```

### 新增收藏地點
```
POST http://localhost:8080/api/places
Content-Type: application/json

{
  "placeId": "ChIJN1t_tDeuEmsRUsoyG83frY4",
  "name": "台北 101",
  "address": "台北市信義區信義路五段7號",
  "latitude": 25.0338,
  "longitude": 121.5645,
  "note": "必去景點",
  "category": "attraction"
}
```

### 附近搜尋餐廳
```
GET http://localhost:8080/api/maps/nearby?lat=25.0338&lng=121.5645&radius=500&type=restaurant
```

### 路線規劃
```
GET http://localhost:8080/api/maps/directions?origin=台北車站&destination=台北101&mode=driving
```

## Docker 常用指令

```bash
# 啟動
docker compose up -d

# 停止（資料保留）
docker compose stop

# 停止並移除容器（資料仍在 volume）
docker compose down

# 完全清除，包含所有資料
docker compose down -v

# 查看 PostgreSQL log
docker compose logs postgres
```

## 部署到雲端

部署至 GCP / AWS / Railway 等平台時，只需修改 `application.properties` 的資料庫連線即可，所有 Java 程式碼不需更動：

```properties
spring.datasource.url=jdbc:postgresql://YOUR_CLOUD_HOST:5432/mapapp_db
spring.datasource.username=YOUR_USER
spring.datasource.password=YOUR_PASSWORD
spring.jpa.hibernate.ddl-auto=update
```

## 專案結構

```
spring-map-app/
├── pom.xml                          ← Maven 依賴（含 PostgreSQL driver）
├── docker-compose.yml               ← PostgreSQL Docker 設定
├── 架構規劃.md                       ← 詳細架構文件
├── 快速啟動指南.md                    ← 本文件
└── src/main/
    ├── java/com/example/mapapp/
    │   ├── MapAppApplication.java    ← 入口點
    │   ├── config/
    │   │   ├── SecurityConfig.java  ← Google OAuth 設定
    │   │   └── MapsApiConfig.java   ← Maps WebClient 設定
    │   ├── model/                   ← 資料庫實體 (JPA Entity)
    │   │   ├── User.java
    │   │   ├── SavedPlace.java
    │   │   └── Route.java
    │   ├── repository/              ← 資料庫存取 (JPA Repository)
    │   ├── service/                 ← 業務邏輯
    │   │   ├── UserService.java
    │   │   ├── PlaceService.java
    │   │   ├── RouteService.java
    │   │   └── MapsApiService.java  ← Google Maps API 呼叫
    │   ├── controller/              ← REST API
    │   │   ├── AuthController.java
    │   │   ├── PlaceController.java
    │   │   ├── RouteController.java
    │   │   └── MapsController.java
    │   ├── dto/                     ← 請求/回應物件
    │   └── security/
    │       └── CustomOAuth2UserService.java  ← Google 登入後同步用戶
    └── resources/
        ├── application.properties   ← 設定檔
        └── static/
            └── index.html           ← 登入首頁
```
