# Spring Map App — 架構規劃文件

## 一、App 定位
一個結合 Google 地圖的個人地點管理 App，功能包含：
- **地點收藏 / 旅遊筆記**：標記、儲存、備註地點
- **路線規劃**：起訖點輸入，呼叫 Google Directions API
- **附近搜尋**：搜尋附近餐廳、景點，呼叫 Google Places API
- **Google 登入**：OAuth 2.0，免自建帳號系統

## 三、資料庫說明（PostgreSQL + Docker）

使用 PostgreSQL 16，透過 Docker Compose 在本機一鍵啟動，資料持久存於 Docker Volume。

```bash
# 啟動資料庫
docker compose up -d

# 確認狀態
docker compose ps
```

**連線資訊：**
| 項目 | 值 |
|------|-----|
| Host | `localhost` |
| Port | `5432` |
| Database | `mapapp_db` |
| Username | `postgres` |
| Password | `mapapp1234` |

**application.properties 設定：**
```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/mapapp_db
spring.datasource.username=postgres
spring.datasource.password=mapapp1234
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=update
```

資料存在 Docker Volume `mapapp_pgdata`，重啟容器後不會消失。
可用 **pgAdmin** 或 **TablePlus** 用上述連線資訊連入查看資料表。

## 五、資料庫 Schema

> Hibernate 會依據 JPA Entity 自動建立資料表，以下為對應的 PostgreSQL Schema 參考。

### users 表
```sql
CREATE TABLE users (
    id          BIGSERIAL PRIMARY KEY,
    google_id   VARCHAR(100) UNIQUE NOT NULL,
    email       VARCHAR(200) NOT NULL,
    name        VARCHAR(100),
    avatar_url  VARCHAR(500),
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### saved_places 表（收藏地點）
```sql
CREATE TABLE saved_places (
    id          BIGSERIAL PRIMARY KEY,
    user_id     BIGINT REFERENCES users(id),
    place_id    VARCHAR(200),
    name        VARCHAR(200) NOT NULL,
    address     VARCHAR(500),
    latitude    DOUBLE PRECISION,
    longitude   DOUBLE PRECISION,
    note        TEXT,
    category    VARCHAR(50),
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### routes 表（路線記錄）
```sql
CREATE TABLE routes (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT REFERENCES users(id),
    name            VARCHAR(200),
    origin          VARCHAR(500),
    destination     VARCHAR(500),
    waypoints       TEXT,
    travel_mode     VARCHAR(20),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 六、API 端點設計

### 認證
| Method | Path | 說明 |
|--------|------|------|
| GET | `/oauth2/authorization/google` | 觸發 Google 登入 |
| GET | `/login/oauth2/code/google` | OAuth callback (Spring 自動處理) |
| GET | `/api/auth/me` | 取得當前登入使用者資訊 |
| POST | `/logout` | 登出 |

### 地點收藏
| Method | Path | 說明 |
|--------|------|------|
| GET | `/api/places` | 取得我的收藏清單 |
| POST | `/api/places` | 新增收藏地點 |
| PUT | `/api/places/{id}` | 更新備註/分類 |
| DELETE | `/api/places/{id}` | 刪除收藏 |

### Google Maps 串接
| Method | Path | 說明 |
|--------|------|------|
| GET | `/api/maps/nearby?lat=&lng=&type=` | 附近搜尋 (Places API) |
| GET | `/api/maps/directions?origin=&destination=` | 路線規劃 (Directions API) |
| GET | `/api/maps/place/{placeId}` | 地點詳情 |

### 路線記錄
| Method | Path | 說明 |
|--------|------|------|
| GET | `/api/routes` | 取得我的路線清單 |
| POST | `/api/routes` | 儲存路線 |
| DELETE | `/api/routes/{id}` | 刪除路線 |

## 七、Google API Key 申請步驟

1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 建立新專案
3. 啟用以下 API：
   - **Maps JavaScript API**（前端地圖顯示）
   - **Places API**（附近搜尋）
   - **Directions API**（路線規劃）
   - **Geocoding API**（地址轉座標）
4. 建立 **OAuth 2.0 用戶端 ID**：
   - 應用程式類型：Web 應用程式
   - 授權的重新導向 URI：`http://localhost:8080/login/oauth2/code/google`
5. 建立 **API Key**（用於後端呼叫 Maps API）

## 八、環境變數設定

```properties
# Google OAuth
spring.security.oauth2.client.registration.google.client-id=YOUR_CLIENT_ID
spring.security.oauth2.client.registration.google.client-secret=YOUR_CLIENT_SECRET

# Google Maps API Key
google.maps.api-key=YOUR_MAPS_API_KEY

# PostgreSQL (Docker)
spring.datasource.url=jdbc:postgresql://localhost:5432/mapapp_db
spring.datasource.username=postgres
spring.datasource.password=mapapp1234
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=update
```

建議透過 Shell 環境變數注入敏感資訊，避免金鑰直接寫入檔案：
```bash
export GOOGLE_CLIENT_ID=...
export GOOGLE_CLIENT_SECRET=...
export GOOGLE_MAPS_API_KEY=...
export DB_PASSWORD=mapapp1234
```
